## 1. Principes Généraux

L'interface de SimGolf est **100% custom paint**, sans aucun contrôle Windows standard :
- **Rendu :** Tous les éléments sont des sprites PCX chargés via jgld.dll
- **Input :** `GetAsyncKeyState()` pour le clavier, hit-test de coordonnées pour la souris
- **Pas de WM_COMMAND** — les boutons sont des rectangles avec des états (normal/survolé/enfoncé)
- **Overlay :** Les infoscreens utilisent black.pcx + masque alpha *pcx + image principale
- **Compositing :** BitBlt/StretchBlt assemble les calques dans le DC final

## 3. Éléments d'Interface

### Boutons

Les boutons sont définis par des zones rectangulaires et ont 3 états visuels :

```
État Normal   → PCX normal (ex: TitleUnSel)
État Survolé  → PCX hover (ex: TitleMO)
État Enfoncé  → PCX pressed (généré dynamiquement?)
```

Structure d'un bouton :
```c
typedef struct {
    int x, y, w, h;           // Rectangle
    int id;                   // Identifiant
    char* label;              // Texte (ou NULL)
    char* pcxNormal;          // Sprite normal
    char* pcxHover;           // Sprite survolé
    void (*onClick)(void*, int);  // Callback
    int soundId;              // Son button1/button2/button3.wav
} ButtonDef;
```

### Popups

Les popups sont des overlays temporaires :
- `PopUpIcons.pcx` + `_A` — Icônes de confirmation
- `Pop_upOK.pcx` + `_A` — Bouton OK
- `TransPopups.pcx` + `_A` — Animations de transition

### Marqueurs et Outils

- `TacksandArrow` + `_A` — Marqueurs de position, flèches directionnelles
- `tacs&tees` + `_A` — Tés et marqueurs de trou
- `3mainLowerLeft` + `_A` — Zone inférieure gauche

## 5. Gestion des Entrées

### InputHandler @ 0x49b7b0

6 types d'événements :

| ID | Nom | Action |
|:--:|-----|--------|
| 0x31 (49) | ESC / Annuler | `vtable_Cancel()` → retour état précédent |
| 3 | Activation golfeur | `StartGolferAction(id, 1)` |
| 5 | Clic terrain | Hit-test → action terrain |
| 0x102 | Sélection golfeur | Cherche slot par ID |
| 0x103 | Bouton interface | Action interface via vtable[0x10] |

Les événements multi-touch (stride 0x28) sont supportés pour les sous-événements.

### Curseurs

```
CURSOR_DEFAULT  = IDC_ARROW (flèche)
CURSOR_INFO     = IDC_CROSS (info bulle) — "Info Cursor"
CURSOR_BUILD    = IDC_HAND  (construction)
CURSOR_ELEVATE  — élévation de terrain
CURSOR_PLANT    — plantation d'arbres
CURSOR_DELETE   — suppression
CURSOR_PAINT    — peinture de terrain
```
